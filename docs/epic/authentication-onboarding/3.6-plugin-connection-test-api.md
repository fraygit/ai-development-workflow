# Task 3.6 — Plugin Connection Test API Endpoints (Backend)

**Story:** 3.0 — Onboarding Plugin Setup Wizard  
**Skill:** backend-lead  
**Priority:** P0 — depends on Tasks 2.3, 3.1  

---

## What to do

Implement the Coordination API endpoints that test plugin credentials by making a real call to the external service (GitHub, GitLab, Jira, Anthropic, Azure). Credentials are proxied through the backend — never called directly from the browser.

---

## Acceptance Criteria

**Endpoints (all require auth)**
- [ ] `POST /plugins/github/test` — body: `{ installationId: string }`. Calls GitHub API to verify the app installation has repo access.
- [ ] `POST /plugins/gitlab/test` — body: `{ groupToken: string, groupNamespace: string }`. Calls GitLab API to list repos in the namespace.
- [ ] `POST /plugins/jira/test` — body: `{ baseUrl, apiToken, userEmail, projectKey }`. Calls `GET {baseUrl}/rest/api/3/myself` with Basic Auth.
- [ ] `POST /plugins/ai/test` — body: `{ provider: 'anthropic' | 'azure', ...providerFields }`. Makes a minimal inference call.

**Response contract**
- [ ] Success: `200 { success: true }`.
- [ ] Failure: `200 { success: false, message: "Plain-language reason" }`. (Return 200, not 4xx — the test itself succeeded; it's the credentials that failed.)
- [ ] Internal error (API unreachable, timeout): `500 { code: "TEST_FAILED", message: "Could not reach the service. Please try again." }`.

**No persistence before success**
- [ ] Connection test endpoints do NOT store credentials or update `tenant_plugins`. Credential storage is in Task 3.7, triggered only after a passing test from the wizard "Next/Finish" button.

**Testing**
> **TDD:** Write each test case below _before_ implementing the handlers. Run with `npx vitest run`. Confirm they fail (Red), implement to pass (Green), then refactor.
- [ ] Unit tests for each test handler using mocked HTTP clients.
- [ ] Test: invalid Jira token → `{ success: false, message: "..." }`.
- [ ] Test: network timeout → `500`.

---

## Notes

- Use `node-fetch` or the native `fetch` (Node 18+) for outbound HTTP calls — no Axios needed.
- Set a 10-second timeout on all outbound test calls to prevent slow external services from blocking the request.
- Log test failures at `warn` level with the tenant ID but WITHOUT the credential values.
