# Task 1.6 — Tenant Creation Service (Backend)

**Story:** 1.0 — Tenant Sign-up & Org Creation  
**Skill:** backend-lead  
**Priority:** P0 — depends on Task 1.2  

---

## What to do

Implement the service function called by the webhook handler (Task 1.5) that creates the `tenants` row with trial defaults.

---

## Acceptance Criteria

**Tenant creation**
- [x] `createTenant({ clerkOrgId, name })` function:
  - Derives `slug` from `name` in kebab-case (`toSlug` strips non-alphanumeric, lowercases).
  - If `slug` already exists, appends a numeric suffix (`myorg-2`, `myorg-3`, ...).
  - Inserts `tenants` row with `region = 'au'`, `plan = 'trial'`, `planStatus = 'active'`, `trialEndsAt = now() + 14 days`.
  - Idempotent: if `clerkOrgId` already exists, returns without inserting.
- [x] `upsertTenantUser({ clerkOrgId, clerkUserId, role })` — looks up tenant by `clerkOrgId`, upserts `tenant_users` row (on conflict: updates `role`, clears `deletedAt`).
- [x] `softDeleteTenantUser({ clerkOrgId, clerkUserId })` — sets `deletedAt = now()`.

**Admin user creation**
- [x] Both user functions resolve `tenantId` from `clerkOrgId` via repository — no direct FK dependency in callers.

**Testing**
> **TDD:** Tests written before implementation. Red confirmed (file not found error), then Green.
- [x] Unit test: `createTenant` with new org → correct defaults (`plan`, `region`, `planStatus`, `trialEndsAt` ±14 days).
- [x] Unit test: `createTenant` same `clerkOrgId` twice → `create` not called second time (idempotent).
- [x] Unit test: slug collision → appends `-3` suffix after two taken slugs.
- [x] Unit test: org name with spaces/special chars → correct kebab-case slug.
- [x] Unit test: `upsertTenantUser` → `upsertUser` called with resolved `tenantId`.
- [x] Unit test: `softDeleteTenantUser` → `softDeleteUser` called with resolved `tenantId`.

## Notes

- Keep this as a pure service function (no Fastify dependency) so it can be unit tested in isolation with a mocked Prisma client.
- `trialEndsAt` should be computed as `new Date(Date.now() + 14 * 24 * 60 * 60 * 1000)`.
