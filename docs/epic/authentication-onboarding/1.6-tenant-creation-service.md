# Task 1.6 — Tenant Creation Service (Backend)

**Story:** 1.0 — Tenant Sign-up & Org Creation  
**Skill:** backend-lead  
**Priority:** P0 — depends on Task 1.2  

---

## What to do

Implement the service function called by the webhook handler (Task 1.5) that creates the `tenants` row with trial defaults.

---

## Acceptance Criteria

**Tenant creation**
- [ ] `createTenant(clerkOrgId, orgName)` function:
  - Derives `slug` from `orgName` in kebab-case.
  - If `slug` already exists, appends a numeric suffix (`myorg-2`, `myorg-3`, ...).
  - Inserts `tenants` row with:
    - `region = 'au'`
    - `plan = 'trial'`
    - `planStatus = 'active'`
    - `trialEndsAt = now() + 14 days` (UTC)
  - Is idempotent: if a row with `clerkOrgId` already exists, returns the existing row without inserting a duplicate.
- [ ] Returns the created (or existing) `tenants` row.

**Admin user creation**
- [ ] `createTenantUser(tenantId, clerkUserId, role)` function:
  - Upserts a `tenant_users` row.
  - On conflict `(tenantId, idpUserId)`, updates `role` and clears `deletedAt`.

**Testing**
> **TDD:** Write each test case below _before_ implementing the service. Run with `npx vitest run`. Confirm they fail (Red), implement to pass (Green), then refactor.
- [ ] Unit test: `createTenant` with a new org name creates row with correct defaults.
- [ ] Unit test: `createTenant` called twice with the same `clerkOrgId` returns existing row, no duplicate.
- [ ] Unit test: slug collision appends suffix correctly.

---

## Notes

- Keep this as a pure service function (no Fastify dependency) so it can be unit tested in isolation with a mocked Prisma client.
- `trialEndsAt` should be computed as `new Date(Date.now() + 14 * 24 * 60 * 60 * 1000)`.
