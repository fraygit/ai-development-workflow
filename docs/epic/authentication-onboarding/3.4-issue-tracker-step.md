# Task 3.4 — Wizard Step 2: Issue Tracker (Frontend)

**Story:** 3.0 — Onboarding Plugin Setup Wizard  
**Skill:** frontend-lead  
**Priority:** P0 — depends on Task 3.2  

---

## What to do

Build the Issue Tracker step form (Step 2 of the wizard) for Jira, with connection test integration.

---

## Acceptance Criteria

**Form fields**
- [ ] Provider radio: "Jira" (only option in MVP, but render as a radio so others can be added later).
- [ ] Required fields:
  - "Jira base URL" — text input, placeholder `https://mycompany.atlassian.net`.
  - "Jira API token" — masked text input.
  - "Jira user email" — email input.
  - "Jira project key" — text input, placeholder `PROJ`, auto-uppercased.
- [ ] "Advanced" toggle reveals:
  - "Webhook secret" — optional masked text input, helper text: "Used to verify inbound Jira webhooks. Leave blank to skip webhook verification."
- [ ] Client-side validation: all required fields must be non-empty before "Test connection" is enabled.

**Connection test**
- [ ] "Test connection" button calls `POST /plugins/jira/test` (Task 3.6) with the form values.
- [ ] On success: "Connected ✓" inline message; "Next" activates; "Skip" is hidden (test passed, no need to skip).
- [ ] On failure: inline red error message from API response.

**Accessibility**
- [ ] All fields labelled, keyboard-navigable, focus moves to first field on step entry.

---

## Notes

- The Jira API token is different from a password — link to Atlassian's docs for creating an API token in the field helper text: "Create a token at id.atlassian.com/manage-profile/security/api-tokens".
- Do not send the API token in a URL query parameter — always POST body.
