## What to do

Define the `tenants` and `tenant_users` Prisma models, generate the initial migration, and verify the schema applies cleanly to the local PostgreSQL instance from Task 1.1.

---

## Acceptance Criteria

**Prisma models**
- [ ] `tenants` model includes:
  - `id` (UUID, primary key, default `uuid()`)
  - `slug` (String, unique) — kebab-case org name
  - `clerkOrgId` (String, unique) — Clerk organisation ID
  - `region` (String, default `'au'`)
  - `plan` (String, default `'trial'`)
  - `planStatus` (String, default `'active'`) — `'active' | 'expired'`
  - `trialEndsAt` (DateTime)
  - `onboardingState` (Json, nullable) — wizard progress (Story 3.0)
  - `createdAt` / `updatedAt` timestamps
- [ ] `tenant_users` model includes:
  - `id` (UUID, primary key)
  - `tenantId` (FK → tenants.id)
  - `idpUserId` (String) — Clerk `userId`
  - `role` (String) — `'admin' | 'engineer' | 'viewer'`
  - `deletedAt` (DateTime, nullable) — soft delete
  - `createdAt` / `updatedAt` timestamps
  - Unique constraint on `(tenantId, idpUserId)`

**Migration**
- [ ] `npx prisma migrate dev --name init-tenants` runs without errors against local PostgreSQL.
- [ ] `npx prisma studio` opens and shows both tables.

**RLS note**
- [ ] A comment in the schema file notes that Row Level Security policies are applied via a separate migration (deferred — GCP/staging concern, not local dev).

**TDD — Integration Tests**
> Write these tests _before_ finalising the schema. Run with `npx vitest run` against the `aidevflow_test` database. Confirm they fail first (Red), apply the migration to pass them (Green).

- [ ] Test: insert a `tenants` row with all required fields → row exists with correct defaults (`plan = 'trial'`, `region = 'au'`).
- [ ] Test: insert two `tenants` rows with the same `clerkOrgId` → unique constraint error thrown.
- [ ] Test: insert two `tenants` rows with the same `slug` → unique constraint error thrown.
- [ ] Test: insert a `tenant_users` row referencing a valid `tenantId` → row created successfully.
- [ ] Test: insert a `tenant_users` row with a non-existent `tenantId` → foreign key constraint error thrown.
- [ ] Test: insert duplicate `(tenantId, idpUserId)` → unique constraint error thrown.
- [ ] A `aidevflow_test` database is created locally for test isolation: `createdb aidevflow_test`. Test suite runs `prisma migrate deploy` against it in setup.
