# Task 3.2 — Wizard UI Shell (Frontend)

**Story:** 3.0 — Onboarding Plugin Setup Wizard  
**Skill:** frontend-lead  
**Priority:** P0 — depends on Tasks 1.7, 3.1  

---

## What to do

Build the outer shell of the plugin setup wizard: step indicator, navigation (back/next), and the Zustand store for unsaved form state.

---

## Acceptance Criteria

**Route & layout**
- [ ] `app/(dashboard)/onboarding/page.tsx` renders the wizard shell.
- [ ] The page is a Server Component that fetches `GET /onboarding` to determine the current step and completed steps. Resuming from the first incomplete step on page load.
- [ ] If onboarding is already complete (`isOnboardingComplete` = true), the page redirects to `/dashboard`.

**Step indicator**
- [ ] A numbered step indicator shows: "Step X of 3" with step names (Source Control / Issue Tracker / AI Provider).
- [ ] Completed steps show a checkmark. Current step is highlighted. Future steps are dimmed.
- [ ] Built with shadcn/ui components (or a simple custom component if a `Steps` primitive isn't in the library yet).

**Navigation**
- [ ] "Next" button advances to the next step. Disabled until the current step's connection test passes.
- [ ] "Back" button returns to the previous step. Always enabled (except on step 1).
- [ ] "Skip" button available on steps 2 and 3 only. Calls `PATCH /onboarding/step/:stepName` with `skipped: true` then advances.

**Zustand store**
- [ ] A `useOnboardingStore` Zustand store holds unsaved form field values for the current step.
- [ ] Store is reset when the wizard unmounts (not persisted to localStorage — server state is the source of truth).

---

## Notes

- Step content (the actual plugin forms) is implemented in Tasks 3.3–3.5; this task is only the shell that wraps them.
- The "Next" button's disabled state is controlled by a `stepValid` boolean passed up from the step component via the Zustand store or a React state callback.
