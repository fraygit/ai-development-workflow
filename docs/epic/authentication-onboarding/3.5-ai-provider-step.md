# Task 3.5 — Wizard Step 3: AI Provider (Frontend)

**Story:** 3.0 — Onboarding Plugin Setup Wizard  
**Skill:** frontend-lead  
**Priority:** P0 — depends on Task 3.2  

---

## What to do

Build the AI Provider step form (Step 3 of the wizard) for Anthropic and Azure AI Foundry, with connection test integration.

---

## Acceptance Criteria

**Form fields**
- [ ] Provider radio: "Anthropic" / "Azure AI Foundry".
- [ ] **Anthropic selected:** Single masked "API key" input, placeholder `sk-ant-...`.
- [ ] **Azure AI Foundry selected:**
  - "Endpoint URL" — text input.
  - "API key" — masked text input.
  - "Deployment name" — text input (the model deployment name in Azure).
- [ ] Provider switch clears all fields and the previous connection test result.
- [ ] Client-side validation: all visible required fields non-empty before test is enabled.

**Connection test**
- [ ] "Test connection" calls `POST /plugins/ai/test` (Task 3.6).
- [ ] On success: "Connection verified ✓" message; "Finish" button activates (this is the last step, so "Next" label changes to "Finish").
- [ ] On failure: inline error message from API.

**Completion**
- [ ] Clicking "Finish" (after a passing test) calls `PATCH /onboarding/step/aiProvider` with `complete: true`, then navigates to the wizard completion screen (Task 3.8).

**Accessibility**
- [ ] Keyboard-navigable; focus moves to first field on step entry.

---

## Notes

- The Anthropic connection test makes a minimal inference call (`claude-haiku-4-5` with a single-token prompt) — inform the user this will incur a minimal API charge in the helper text.
- Azure requires endpoint + key + deployment name; validate all three are non-empty before enabling the test.
