# Task 4.3 — Usage Meter UI (Frontend)

**Story:** 4.0 — Trial Activation & Status Display  
**Skill:** frontend-lead  
**Priority:** P0 — depends on Tasks 2.1, 4.4  

---

## What to do

Build the usage meter on the Settings → Billing page showing task execution consumption vs. plan limits for the current billing cycle.

---

## Acceptance Criteria

**Page location**
- [ ] `app/(dashboard)/settings/billing/page.tsx` — accessible to all roles (read-only display); the "Upgrade" action visible to Admin only.

**Usage meter**
- [ ] Fetches `GET /usage/current-cycle` from the Coordination API.
- [ ] Displays a progress bar showing `tasksUsed / tasksLimit` with label "X of Y task executions used".
- [ ] Breakdown by workflow name listed beneath the progress bar: each row shows workflow name and execution count.
- [ ] When `tasksUsed >= tasksLimit`, the progress bar turns red and an inline warning shows "You have reached your task execution limit."

**Refresh behaviour**
- [ ] Page refetches usage data when it gains focus (TanStack Query `refetchOnWindowFocus: true`).
- [ ] A 60-second polling interval keeps the meter near-real-time while the page is open.

**Trial plan info**
- [ ] The billing page shows the current plan name ("Trial"), trial expiry date, and the "Upgrade" CTA (Admin only) above the usage meter.

---

## Notes

- The usage data endpoint (`GET /usage/current-cycle`) is implemented in Task 4.4.
- Use shadcn/ui `Progress` component for the progress bar.
- If the backend API is not yet available, show a skeleton/loading state — do not show stale zeros.
