## What to do

| Concern | Repo |
|---|---|
| Docker Compose (PostgreSQL + Redis) | `ai-development-workflow-backend` |

## GitFlow Convention (both repos)

| Branch type | Pattern | Branch from | Merges into |
|---|---|---|---|
| Feature | `feature/<task-id>-<short-title>` | `develop` | `develop` |
| Release | `release/<semver>` | `develop` | `main` + `develop` |
| Hotfix | `hotfix/<description>` | `main` | `main` + `develop` |

Branch protection rules (to enforce in GitHub settings):
- `main`: PR required, 2 reviewers, no direct push, status checks must pass.
- `develop`: PR required, 1 reviewer, no direct push.

## Notes

- PostgreSQL and Redis are run via Docker Compose for local dev only. GCP Cloud SQL and Memorystore are used in staging/prod.
- The backend API runs on `localhost:3001`. The web app runs on `localhost:3000`.
- Do not commit `.env` or `.env.local` ‚Äî only the `.example` files.
