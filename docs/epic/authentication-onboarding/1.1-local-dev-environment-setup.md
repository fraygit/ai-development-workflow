# Task 1.1 — Local Dev Environment Setup

**Story:** 1.0 — Tenant Sign-up & Org Creation  
**Skill:** local-setup  
**Priority:** P0 — prerequisite for all other tasks  

---

## What to do

Set up the local development environment so all stories in this epic can be worked on without GCP infrastructure. Covers Node.js version, Docker-based PostgreSQL and Redis, and environment variable scaffolding for the web repo.

---

## Acceptance Criteria

**Node.js**
- [ ] Confirm Node.js version is 20 LTS (`node -v`). If < 20, install via `nvm` or download from nodejs.org.
- [ ] `npm -v` confirms npm 10+ is available.
- [ ] `.nvmrc` file added to the web repo root with content `20` (if nvm is used).

**Docker / local services**
- [ ] `docker-compose.yml` added to the web repo root (or a top-level `local/` folder) with:
  - `postgres:16` service — port `5432`, db `aidevflow`, user `aidevflow`, password `aidevflow`.
  - `redis:7` service — port `6379`.
- [ ] `docker compose up -d` starts both services cleanly.
- [ ] PostgreSQL is reachable: `psql postgresql://aidevflow:aidevflow@localhost:5432/aidevflow -c '\l'` returns the db list.

**Environment variables (web repo)**
- [ ] `.env.local.example` file added to the web repo with all required keys stubbed out (no real secrets):
  ```
  NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_test_...
  CLERK_SECRET_KEY=sk_test_...
  CLERK_WEBHOOK_SECRET=whsec_...
  DATABASE_URL=postgresql://aidevflow:aidevflow@localhost:5432/aidevflow
  NEXT_PUBLIC_API_URL=http://localhost:3001
  ```
- [ ] `.env.local` is gitignored (verify `.gitignore` includes it).
- [ ] Developer copies `.env.local.example` → `.env.local` and fills in Clerk keys from Task 1.3.

**Dependencies**
- [ ] `npm install` runs without errors in the web repo.
- [ ] `npm run dev` starts the Next.js dev server on port 3000.

---

## Notes

- PostgreSQL and Redis are run via Docker Compose for local dev only. GCP Cloud SQL and Memorystore are used in staging/prod (deferred — not part of this task).
- The backend API repo does not exist yet. When it is created, it will also need `DATABASE_URL` and `REDIS_URL` pointing at the same local Docker services.
- Do not commit `.env.local` — only `.env.local.example`.
