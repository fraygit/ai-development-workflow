## What to do

Add the Clerk-hosted sign-up page to the Next.js web app and wire the "Get started" CTA on the marketing pages to it.

---

## Acceptance Criteria

**Clerk Next.js SDK**
- [x] `@clerk/nextjs` installed in the web repo.
- [x] `ClerkProvider` wraps the root layout (`app/layout.tsx`).
- [x] `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` is consumed by the provider (auto-picked from env).

**Sign-up route**
- [x] `app/(auth)/sign-up/[[...sign-up]]/page.tsx` created using the `<SignUp />` component from `@clerk/nextjs`.
- [x] `NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up` and `NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/dashboard` added to `.env.local.example` (redirect to `/dashboard` — placeholder until onboarding wizard is built in Story 3.0).
- [x] After successful sign-up Clerk redirects to `/dashboard`.

**Marketing page CTA**
- [x] `app/page.tsx` replaced with AIDevFlow landing page — "Get started" links to `/sign-up`.
- [x] CTA visible without authentication (middleware does not protect root route).

**Error display**
- [x] Clerk's built-in error messages shown within `<SignUp />` — no custom handling needed.

## Notes

- Use Clerk's prebuilt `<SignUp />` component — do not build a custom form.
