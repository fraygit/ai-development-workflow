# Task 1.4 — Clerk Sign-up Page (Frontend)

**Story:** 1.0 — Tenant Sign-up & Org Creation  
**Skill:** frontend-lead  
**Priority:** P0  

---

## What to do

Add the Clerk-hosted sign-up page to the Next.js web app and wire the "Get started" CTA on the marketing pages to it.

---

## Acceptance Criteria

**Clerk Next.js SDK**
- [ ] `@clerk/nextjs` installed in the web repo.
- [ ] `ClerkProvider` wraps the root layout (`app/layout.tsx`).
- [ ] `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` is consumed by the provider.

**Sign-up route**
- [ ] `app/(auth)/sign-up/[[...sign-up]]/page.tsx` created using the `<SignUp />` component from `@clerk/nextjs`.
- [ ] `NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up` and `NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL=/onboarding` set in `.env.local`.
- [ ] After successful sign-up, Clerk redirects to `/onboarding` (wizard — Story 3.0). If wizard is not yet implemented, redirect to `/dashboard` as a placeholder.

**Marketing page CTA**
- [ ] A "Get started" button/link on the public marketing page (`app/(marketing)/page.tsx` or equivalent) links to `/sign-up`.
- [ ] The CTA is visible without authentication.

**Error display**
- [ ] Clerk's built-in error messages are shown within the `<SignUp />` component — no custom error handling needed at this stage.

---

## Notes

- Use Clerk's prebuilt `<SignUp />` component — do not build a custom form.
- The `/onboarding` route (plugin setup wizard) will be implemented in Story 3.0. Use `/dashboard` as the redirect target until then.
- Ensure `middleware.ts` does NOT protect `/sign-up` or the marketing routes (public access required).
