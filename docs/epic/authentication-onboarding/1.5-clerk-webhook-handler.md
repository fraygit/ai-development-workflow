# Task 1.5 — Clerk Webhook Handler (Backend)

**Story:** 1.0 — Tenant Sign-up & Org Creation  
**Skill:** backend-lead  
**Priority:** P0 — depends on Tasks 1.2 and 1.3  

---

## What to do

Implement the `/webhooks/clerk` endpoint in the Coordination API (Fastify) that receives Clerk lifecycle events and triggers tenant row creation.

---

## Acceptance Criteria

**Endpoint**
- [ ] `POST /webhooks/clerk` route registered in Fastify.
- [ ] HMAC-SHA256 signature verified using `CLERK_WEBHOOK_SECRET` before any processing. Invalid signature returns `400` immediately.
- [ ] Use the `svix` npm package (Clerk's webhook library) for signature verification.

**Event handling**
- [ ] `organization.created` event → calls the tenant creation service (Task 1.6).
- [ ] `user.created` event → stored for reference; no action required until `organizationMembership.created`.
- [ ] `organizationMembership.created` event → upserts `tenant_users` row with the role from event metadata (`admin` for the creator, or the role passed in the invitation — Story 5.0).
- [ ] `organizationMembership.deleted` event → sets `tenant_users.deletedAt = now()`.
- [ ] Unknown event types return `200` with no action (do not error on future events).

**Error handling**
- [ ] If DB write fails, the handler returns `500` so Clerk retries the webhook.
- [ ] Duplicate `organization.created` events (Clerk retry) are handled idempotently — the tenant creation service checks for existing `clerkOrgId` before inserting.

**Testing**
- [ ] Unit test: signature verification rejects a request with an incorrect signature.
- [ ] Unit test: `organization.created` payload triggers tenant creation service with correct arguments.
- [ ] Unit test: duplicate `organization.created` does not create a second tenant row.

---

## Notes

- This endpoint lives in the backend Coordination API repo. It is spec'd here so the frontend team knows what to expect when sign-up is tested end-to-end.
- During local dev, use the ngrok tunnel from Task 1.3 to receive real Clerk webhook deliveries.
- Clerk delivers webhooks with a `svix-id`, `svix-timestamp`, and `svix-signature` header. Use the `Webhook` class from the `svix` package to verify.
