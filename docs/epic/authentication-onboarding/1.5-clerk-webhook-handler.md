# Task 1.5 — Clerk Webhook Handler (Backend)

**Story:** 1.0 — Tenant Sign-up & Org Creation  
**Skill:** backend-lead  
**Priority:** P0 — depends on Tasks 1.2 and 1.3  

---

## What to do

Implement the `/webhooks/clerk` endpoint in the Coordination API (Fastify) that receives Clerk lifecycle events and triggers tenant row creation.

---

## Acceptance Criteria

**Endpoint**
- [x] `POST /webhooks/clerk` route registered in Fastify.
- [x] HMAC-SHA256 signature verified using `CLERK_WEBHOOK_SECRET` before any processing. Invalid signature returns `400` immediately.
- [x] `svix` npm package used for signature verification. Raw body captured via custom `addContentTypeParser` so svix receives the original bytes.

**Event handling**
- [x] `organization.created` event → calls `tenantService.createTenant()`.
- [x] `user.created` event → no action (linked on `organizationMembership.created`).
- [x] `organizationMembership.created` event → calls `tenantService.upsertTenantUser()` with role mapped (`org:admin` → `'admin'`, otherwise `'engineer'`).
- [x] `organizationMembership.deleted` event → calls `tenantService.softDeleteTenantUser()`.
- [x] Unknown event types return `200` with no action.

**Error handling**
- [x] DB write failure → `500` so Clerk retries.
- [x] Duplicate `organization.created` handled by the service (Task 1.6) — handler calls through both times; returns `200` in both cases.

**Testing**
- [x] Unit test: invalid signature → `400`, service not called.
- [x] Unit test: `organization.created` → `createTenant` called with correct `clerkOrgId` and `name`.
- [x] Unit test: duplicate `organization.created` → both calls return `200` (service absorbs duplicate).
- [x] Unit test: `organizationMembership.created` → `upsertTenantUser` called with correct args.
- [x] Unit test: `organizationMembership.deleted` → `softDeleteTenantUser` called.
- [x] Unit test: unknown event type → `200`, no service calls.
- [x] Unit test: service throws → `500`.

## Notes

- This endpoint lives in the backend Coordination API repo. It is spec'd here so the frontend team knows what to expect when sign-up is tested end-to-end.
- During local dev, use the ngrok tunnel from Task 1.3 to receive real Clerk webhook deliveries.
- Clerk delivers webhooks with a `svix-id`, `svix-timestamp`, and `svix-signature` header. Use the `Webhook` class from the `svix` package to verify.
