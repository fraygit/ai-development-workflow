# Task 3.1 — Wizard State Schema & API (Backend)

**Story:** 3.0 — Onboarding Plugin Setup Wizard  
**Skill:** backend-lead  
**Priority:** P0 — depends on Task 1.2  

---

## What to do

Define the wizard state structure stored on the `tenants` row and implement the backend API endpoints to read and update onboarding progress.

---

## Acceptance Criteria

**Schema**
- [ ] `tenants.onboardingState` (JSON) stores:
  ```json
  {
    "currentStep": 1,
    "steps": {
      "sourceControl": { "complete": false, "skipped": false },
      "issueTracker":  { "complete": false, "skipped": false },
      "aiProvider":    { "complete": false, "skipped": false }
    }
  }
  ```
- [ ] Helper: `isOnboardingComplete(state)` returns `true` if all steps are `complete: true` or `skipped: true` (except `sourceControl` — it cannot be skipped per Story 3.0 AC).

**API endpoints**
- [ ] `GET /onboarding` → returns the current `onboardingState` for the authenticated tenant.
- [ ] `PATCH /onboarding/step/:stepName` → updates a step's `complete` or `skipped` flag. Validates `stepName` is one of `sourceControl | issueTracker | aiProvider`.
- [ ] Both endpoints require auth (Task 2.3 plugin).

**tenant_plugins table**
- [ ] `tenant_plugins` Prisma model:
  - `id` (UUID)
  - `tenantId` (FK → tenants.id)
  - `pluginType` (String) — `'github' | 'gitlab' | 'jira' | 'anthropic' | 'azure'`
  - `enabled` (Boolean, default `false`)
  - `secretPath` (String, nullable) — path in Secret Manager (deferred for GCP; use env var path for local dev)
  - Unique constraint `(tenantId, pluginType)`
  - `createdAt` / `updatedAt`

---

## Notes

- `sourceControl` cannot be skipped — the backend should return `400` if `PATCH /onboarding/step/sourceControl` is called with `skipped: true`.
- For local dev, `secretPath` can store a reference like `LOCAL:JIRA_API_TOKEN` pointing to a `.env.local` key instead of GCP Secret Manager.
- Prisma migration: `npx prisma migrate dev --name add-tenant-plugins`.
