# Task 1.7 — Post-signup Redirect to Wizard (Frontend)

**Story:** 1.0 — Tenant Sign-up & Org Creation  
**Skill:** frontend-lead  
**Priority:** P0 — depends on Task 1.4  

---

## What to do

Ensure that after a successful sign-up, the user lands on the plugin setup wizard (Story 3.0) as their first dashboard screen, not the main dashboard.

---

## Acceptance Criteria

- [ ] `NEXT_PUBLIC_CLERK_AFTER_SIGN_UP_URL` is set to `/onboarding` in `.env.local`.
- [ ] `app/(dashboard)/onboarding/page.tsx` exists (can be a placeholder page until Story 3.0 is implemented).
- [ ] The placeholder page is protected by `middleware.ts` (authenticated-only).
- [ ] Navigating to `/onboarding` when the wizard is already complete (all three plugins configured) redirects to `/dashboard` — skip the wizard re-run.
- [ ] The onboarding completion check is done server-side: read `tenant.onboardingState` from the Coordination API; if complete, use `redirect('/dashboard')` in the Server Component.

---

## Notes

- The full wizard implementation is in Story 3.0. This task only sets up routing and the guard logic.
- For now, the placeholder `/onboarding` page can show "Setup in progress..." until the wizard UI is built.
- `onboardingState` on the `tenants` row is set to complete by the backend after all wizard steps pass (Task 3.1).
