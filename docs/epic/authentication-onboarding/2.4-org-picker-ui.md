# Task 2.4 — Org Picker UI (Frontend)

**Story:** 2.0 — Sign In & Session Management  
**Sub-story:** 2.1 — Org Selection  
**Skill:** frontend-lead  
**Priority:** P0 — required for multi-org users  

---

## What to do

Implement an org selection screen shown after sign-in when the user belongs to more than one Clerk organisation, before routing to `/dashboard`.

---

## Acceptance Criteria

- [ ] `app/(auth)/org-selection/page.tsx` created.
- [ ] `NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/org-selection` replaces the direct `/dashboard` redirect (middleware will redirect single-org users past this screen automatically).
- [ ] The page uses `<OrganizationList />` from `@clerk/nextjs` to display the user's organisations with name and role.
- [ ] Selecting an org sets `activeOrganizationId` via Clerk's `setActive()` and redirects to `/dashboard`.
- [ ] If the user has only one org, `middleware.ts` skips the org picker and redirects directly to `/dashboard` (check `orgId` in session — if already set, skip).
- [ ] An "Org switcher" component (Clerk `<OrganizationSwitcher />`) is present in the dashboard top navigation so users can switch orgs without full sign-out.

---

## Notes

- Clerk's `<OrganizationList />` and `<OrganizationSwitcher />` handle the multi-org UX — no custom org-picker UI is needed.
- The org-switcher in the nav re-triggers active org selection, which Next.js middleware then validates on the next route.
