# Story 1.0 — Tenant Sign-up & Org Creation

**Epic:** Authentication & Onboarding  
**Priority:** P0 — MVP critical path  
**Roles involved:** New tenant (admin)

---

## User Story

As a new user visiting AIDevFlow for the first time,  
I want to sign up with my email address and have an organisation automatically created for me,  
so that I can start configuring the platform without any manual provisioning step.

---

## Acceptance Criteria

- [ ] Clerk-hosted sign-up flow is accessible from the public marketing pages via a "Get started" CTA.
- [ ] A new user can sign up with email + password or a supported OAuth provider (Google).
- [ ] On successful sign-up, Clerk creates a new organisation scoped to the user's email domain (or prompted org name if domain-based creation is disabled).
- [ ] The platform's backend creates a `tenants` row with:
  - `region = 'au'` (AU-only for MVP)
  - `plan = 'trial'`
  - `plan_status = 'active'`
  - `trial_ends_at = now() + 14 days`
- [ ] The signing-up user is recorded in `tenant_users` with `role = 'admin'`.
- [ ] On completion, the user is redirected to the dashboard — the plugin setup wizard is the first screen shown (see Story 3.0).
- [ ] If an org with the same slug already exists, Clerk handles the conflict; the user is prompted to join the existing org (invite flow) rather than creating a duplicate.
- [ ] No manual admin action is required to provision the tenant.

---

## Technical Notes

- Clerk handles identity (email verification, OAuth). The platform receives a `user.created` and `organization.created` webhook from Clerk to trigger backend tenant row creation.
- `tenants.slug` is derived from the org name (kebab-case, unique). Collision appends a numeric suffix.
- `tenant_users.idp_user_id` is the Clerk `userId`.
- Region is hard-coded to `au` for MVP. Region selection UI is post-MVP.
- All DB writes scoped with RLS — the service account used by the Coordination API has a `tenant_id` context set at session level.

---

## Dependencies

- Clerk project configured with `organization` feature enabled.
- Clerk webhooks (`user.created`, `organization.created`, `organizationMembership.created`) wired to the Coordination API.
- `tenants` and `tenant_users` Prisma schema and migrations in place (Story Platform Infra).
- GCP Secret Manager bootstrap completed (Platform Infra epic).

---

## Out of Scope for This Story

- SSO / SAML federation (Enterprise tier, post-MVP).
- Region selection at signup (post-MVP).
- Inviting additional users at signup (covered by Story 5.0).
- Plugin configuration (covered by Story 3.0).

---

## Definition of Done

- [ ] Clerk sign-up flow working end-to-end in staging environment.
- [ ] `tenants` and `tenant_users` rows created automatically on first sign-up.
- [ ] New admin user lands on the plugin setup wizard after sign-up.
- [ ] Unit tests cover tenant creation webhook handler (Clerk → API).
- [ ] Integration test: full sign-up flow from public page → dashboard redirect.
- [ ] No raw Clerk errors surfaced to the user — all errors show plain-language messages.
