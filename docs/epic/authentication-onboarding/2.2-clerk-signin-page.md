# Task 2.2 — Clerk Sign-in Page (Frontend)

**Story:** 2.0 — Sign In & Session Management  
**Skill:** frontend-lead  
**Priority:** P0 — depends on Tasks 1.3, 2.1  

---

## What to do

Add the Clerk-hosted sign-in page and wire the "Sign in" link on all public pages to it.

---

## Acceptance Criteria

- [ ] `app/(auth)/sign-in/[[...sign-in]]/page.tsx` created using the `<SignIn />` component from `@clerk/nextjs`.
- [ ] `NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in` and `NEXT_PUBLIC_CLERK_AFTER_SIGN_IN_URL=/dashboard` set in `.env.local`.
- [ ] "Sign in" link visible in the marketing page header, linking to `/sign-in`.
- [ ] On successful sign-in, user lands on `/dashboard` (or the `redirect_url` param if present).
- [ ] Sign-in supports email + password and Google OAuth (Clerk handles both via the `<SignIn />` component).

**Sign-out**
- [ ] A `<SignOutButton>` (or `<UserButton>`) is present in the dashboard navigation.
- [ ] Signing out clears the Clerk session cookie and redirects to `/` (marketing home).
- [ ] Set `NEXT_PUBLIC_CLERK_AFTER_SIGN_OUT_URL=/` in `.env.local`.

---

## Notes

- Use Clerk's prebuilt `<SignIn />` component — do not build a custom form.
- The `<UserButton>` from `@clerk/nextjs` provides sign-out and account management in one component — prefer this over a standalone sign-out button.
