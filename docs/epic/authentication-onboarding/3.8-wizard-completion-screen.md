# Task 3.8 — Wizard Completion Screen (Frontend)

**Story:** 3.0 — Onboarding Plugin Setup Wizard  
**Skill:** frontend-lead  
**Priority:** P0 — depends on Tasks 3.3–3.5  

---

## What to do

Build the "You're all set" completion screen shown after all wizard steps pass (or are skipped where allowed).

---

## Acceptance Criteria

- [ ] Displayed after the last step completes or when `GET /onboarding` returns `isOnboardingComplete = true`.
- [ ] Shows: headline "You're all set", a summary list of which plugins were connected vs. skipped.
- [ ] Two CTAs:
  - "Create your first workflow" → navigates to `/dashboard/workflows/new` (can be a placeholder route for now).
  - "Go to Dashboard" → navigates to `/dashboard`.
- [ ] The "Complete your setup" banner in the dashboard layout (Story 4.0 placeholder) is hidden once onboarding is complete.
- [ ] The completion screen is not shown to returning users who have already completed onboarding — if they navigate to `/onboarding` manually, they are redirected to `/dashboard`.
- [ ] Plugin status summary shows each plugin with a "Connected" (green) or "Skipped" (grey) badge.

---

## Notes

- Onboarding completion state is determined server-side from `tenant.onboardingState` — not a client-side flag.
- The `/dashboard/workflows/new` route does not need to exist yet; a 404 is acceptable at this stage.
