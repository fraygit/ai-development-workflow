# Task 4.1 — Trial Status Banner (Frontend)

**Story:** 4.0 — Trial Activation & Status Display  
**Skill:** frontend-lead  
**Priority:** P0 — depends on Tasks 1.6, 2.1  

---

## What to do

Build the persistent trial status banner displayed in the dashboard layout, with progressive warning styles as the trial approaches expiry.

---

## Acceptance Criteria

**Banner display**
- [ ] Banner renders beneath the top navigation in the dashboard layout for all `plan = 'trial'` and `planStatus = 'active'` tenants.
- [ ] Banner is hidden when `plan` is not `'trial'` or `planStatus` is `'expired'` (expired handled by Story 6.0 modal).

**Content and variants**

| Days remaining | Variant | Text |
|---|---|---|
| > 7 | default | "Trial — X days remaining" |
| ≤ 7 | info (blue) | "Your trial ends in X days. Upgrade to keep full access." |
| ≤ 3 | warning (amber) | "Your trial ends in X days. Upgrade now to avoid interruption." |
| ≤ 1 | danger (red) | "Your trial ends tomorrow." |
| < 1 (same day) | danger (red) | "Your trial expires today." |

- [ ] "Upgrade" CTA link visible only to `role = 'admin'`; links to `/dashboard/settings/billing`.
- [ ] Variant and `daysRemaining` are computed server-side in the dashboard layout Server Component — not in the client.

**Refresh**
- [ ] Banner data is fetched with `revalidate: 60` (stale-while-revalidate). On the expiry day, the layout uses `revalidate: 0` to avoid stale display.

---

## Notes

- Use shadcn/ui `Alert` component with the `variant` prop mapped to the four states above.
- `daysRemaining = Math.ceil((trial_ends_at - now()) / (1000 * 60 * 60 * 24))`. Use server UTC time.
- The banner is a Server Component — no client-side timer or polling needed for the banner itself.
