# Task 1.3 — Clerk Project Configuration

**Story:** 1.0 — Tenant Sign-up & Org Creation  
**Skill:** backend-lead  
**Priority:** P0 — required before Tasks 1.4, 1.5, 2.x  

---

## What to do

Create and configure a Clerk development application so that sign-up, sign-in, and organisation webhooks work locally. This is a one-time manual setup step.

---

## Acceptance Criteria

**Clerk app creation**
- [ ] A new Clerk application is created at [clerk.com](https://clerk.com) for the `AIDevFlow` project.
- [ ] Email + password and Google OAuth are enabled as sign-in methods.
- [ ] Organisations feature is enabled (Clerk dashboard → Organisations → Enable).
- [ ] Automatic organisation creation on sign-up is enabled (or confirm the setting that triggers `organization.created` on sign-up).

**API keys**
- [ ] `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` and `CLERK_SECRET_KEY` copied from Clerk dashboard into `.env.local` in the web repo.
- [ ] `CLERK_WEBHOOK_SECRET` obtained after Step "Webhooks" below and added to `.env.local`.

**Webhooks (local dev with ngrok or Clerk dev tunnels)**
- [ ] A Clerk webhook endpoint is registered pointing to `https://<tunnel-url>/webhooks/clerk`.
- [ ] The following events are subscribed:
  - `user.created`
  - `organization.created`
  - `organizationMembership.created`
  - `organizationMembership.deleted`
- [ ] Webhook signature secret is saved as `CLERK_WEBHOOK_SECRET`.

**Local tunnel**
- [ ] `ngrok http 3001` (or Clerk's built-in dev tunnels) used to expose the local backend port for webhook delivery during development.
- [ ] Tunnel URL documented in a local scratch note (not committed).

---

## Notes

- Use Clerk's **development** environment — never point the production app at localhost.
- Clerk's "Redirect URLs" should include `http://localhost:3000` for local dev sign-in.
- The backend webhook endpoint (`/webhooks/clerk`) is implemented in Task 1.5. Register the Clerk endpoint here; implement the handler there.
- Google OAuth in development may require a Google Cloud Console OAuth app — create one with `http://localhost:3000` as an authorised redirect URI if needed.
