# Story 4.0 — Trial Activation & Status Display

**Epic:** Authentication & Onboarding  
**Priority:** P0 — MVP critical path  
**Roles involved:** Admin, Engineer, Viewer

---

## User Story

As a new tenant on the Trial plan,  
I want to see a clear indicator of my trial status and remaining time,  
so that I understand my current limits and know when I need to upgrade.

---

## Acceptance Criteria

**Trial activation**
- [ ] A 14-day trial is activated automatically when the `tenants` row is created (Story 1.0). No manual admin action is required.
- [ ] `tenants.plan = 'trial'`, `tenants.plan_status = 'active'`, `tenants.trial_ends_at = signup_timestamp + 14 days`.
- [ ] Trial limits apply immediately:
  - Maximum 1 linked repository.
  - No SSO federation.
  - Community support only.
  - Task execution quota as defined for Trial tier.

**Trial status banner**
- [ ] A trial banner is shown persistently in the dashboard layout (beneath the top navigation) while the trial is active.
- [ ] The banner shows: "Trial — X days remaining" where X is calculated from `trial_ends_at - now()`.
- [ ] The banner includes an "Upgrade" CTA link that navigates to the billing upgrade flow.
- [ ] The banner is visible to all roles (Admin, Engineer, Viewer) — it is informational, not actionable for non-admins (upgrade link shown only to Admins).

**Progressive trial warnings**
- [ ] At 7 days remaining: banner changes style to "info" (blue), text: "Your trial ends in 7 days. Upgrade to keep full access."
- [ ] At 3 days remaining: banner changes style to "warning" (amber), text: "Your trial ends in 3 days. Upgrade now to avoid interruption."
- [ ] At 1 day remaining: banner changes style to "danger" (red), text: "Your trial ends tomorrow."
- [ ] Day-of expiry (< 24h remaining): banner text changes to "Your trial expires today."

**Usage meter**
- [ ] The Settings → Billing page shows a usage meter: task executions consumed vs. plan limit for the current billing cycle.
- [ ] Usage is broken down by workflow name.
- [ ] Meter updates in near-real-time (refreshes when the page gains focus or on a 60-second poll interval).

**Trial plan enforcement — limits**
- [ ] Attempting to link a second repository when already at the 1-repo Trial limit shows an inline error: "Your trial plan allows 1 repository. Upgrade to link more."
- [ ] Attempting to enable SSO (if that UI exists) shows a plan gate message.
- [ ] Plan limits are enforced server-side in the Coordination API — client-side UI messages are informational, not the enforcement boundary.

---

## Technical Notes

- `trial_ends_at` is stored in UTC. All comparisons use server-side UTC time — never client local time.
- The dashboard layout Server Component reads `tenants.trial_ends_at` and `tenants.plan_status` from the Coordination API on each request (cached with `revalidate: 60` — stale-while-revalidate, no staleness on trial expiry day).
- Banner style variant (none / info / warning / danger) is computed server-side and passed as a prop to the Client Component banner.
- Task execution counting: every billable task execution increments a counter in Redis (BullMQ job on completion) and is persisted to a `task_execution_usage` table. The usage meter reads from this table.
- Plan enforcement: the Coordination API middleware checks `plan_status` and per-plan limits before processing requests that could exceed them. Returns `403` with a structured error body (`code: "PLAN_LIMIT_EXCEEDED"`, `message: "..."`) on violation.

## Definition of Done

- [ ] Trial banner visible in dashboard for a new trial tenant.
- [ ] Banner text and style change correctly at 7-day, 3-day, and 1-day thresholds.
- [ ] Upgrade CTA visible only to Admin role.
- [ ] Usage meter displays on Settings → Billing page.
- [ ] 1-repo limit enforced server-side; inline error shown on second repo link attempt.
- [ ] Unit tests: plan limit middleware (`PLAN_LIMIT_EXCEEDED` response on limit breach).
- [ ] Unit tests: banner variant computation for each day-threshold.
- [ ] Snapshot test: banner renders correctly at each variant (no / info / warning / danger).
