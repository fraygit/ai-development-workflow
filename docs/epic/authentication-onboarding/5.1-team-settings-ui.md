# Task 5.1 — Team Settings UI (Frontend)

**Story:** 5.0 — User Invitation & Role Assignment  
**Skill:** frontend-lead  
**Priority:** P1  

---

## What to do

Build the Settings → Team page where Admins can view team members, send invitations, change roles, and remove members.

---

## Acceptance Criteria

**Page location**
- [ ] `app/(dashboard)/settings/team/page.tsx` — accessible only to `role = 'admin'` (middleware guard: redirect non-admins to `/dashboard` with a toast "You don't have permission to manage team members").

**Active members list**
- [ ] Fetches `GET /team` — displays each member's name, email, role badge (Admin/Engineer/Viewer), and date joined.
- [ ] Actions per member:
  - "Change role" dropdown (Admin/Engineer/Viewer). Disabled on the current user's own row.
  - "Remove" button. Disabled if the member is the last Admin.
- [ ] Optimistic UI: role change and removal update immediately; revert on API error with an error toast.

**Invite form**
- [ ] Inline form at the top of the page: email input + role selector + "Send invite" button.
- [ ] Multiple rows: an "Add another" link appends a new row before sending.
- [ ] "Send invite" calls `POST /team/invite` for each row. Shows a success toast per invite sent.
- [ ] If an email already has a pending invite, shows inline warning: "Already invited — the existing invite will be replaced."

**Pending invitations list**
- [ ] Section below active members: lists pending invitations with email, role, and date sent.
- [ ] "Resend" and "Revoke" actions per pending invite.

---

## Notes

- Use shadcn/ui `Table`, `Badge`, `Select`, and `Button` components.
- Role change is a `PATCH /team/:userId/role` call; removal is `DELETE /team/:userId`.
- The self-demotion guard is enforced server-side (Task 5.3) — disable the action client-side too for clarity, but do not rely on it as the enforcement boundary.
