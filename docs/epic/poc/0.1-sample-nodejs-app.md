## What to do

Create a minimal Node.js "Hello World" console app in the sample repo. This is the **target repository** that the POC workflow operates on — Claude Code will clone it, make changes to it, and push a PR back to it. The app must be simple enough that any Jira ticket describing a small change is easy to verify.

**Repo:** `ai-development-workflow-sample-nodejs` ([fraygit/ai-development-workflow-sample-nodejs](https://github.com/fraygit/ai-development-workflow-sample-nodejs))

---

## Files to Create

### `package.json`

```json
{
  "name": "ai-development-workflow-sample-nodejs",
  "version": "1.0.0",
  "description": "Sample Node.js app — POC target for AIDevFlow automated code changes",
  "main": "src/index.js",
  "scripts": {
    "start": "node src/index.js",
    "test": "node --test test/**/*.test.js"
  },
  "engines": {
    "node": ">=20"
  }
}
```

Uses Node's built-in `node:test` runner — no test framework to install.

### `src/index.js`

```js
function greet() {
  return 'Hello World'
}

function main() {
  console.log(greet())
}

main()

module.exports = { greet, main }
```

Exporting `greet` lets the test import and assert on it without running `main()`.

### `test/index.test.js`

```js
const { test } = require('node:test')
const assert = require('node:assert/strict')
const { greet } = require('../src/index')

test('greet returns Hello World', () => {
  assert.equal(greet(), 'Hello World')
})
```

## PAT Setup

The POC workflow clones and pushes to this repo using a **Personal Access Token (PAT)**. The `GITHUB_TOKEN` provided by GitHub Actions only has access to the repo the workflow is *running in* (the POC repo) — not to this separate repo.

**Create the PAT:**
1. Go to GitHub → Settings → Developer Settings → Personal Access Tokens → Fine-grained tokens → Generate new token.
2. **Resource owner:** `fraygit`
3. **Repository access:** Only selected repositories → `ai-development-workflow-sample-nodejs`
4. **Permissions:**
   - `Contents` → Read and write (clone, push branches)
   - `Pull requests` → Read and write (create PRs)
5. Set expiry to a sensible value (90 days for the POC).
6. Copy the token — it is only shown once.

**Store the PAT as a secret in the POC repo:**
1. Go to `ai-development-workflow-poc` → Settings → Secrets and variables → Actions → New repository secret.
2. Name: `SAMPLE_NODEJS_PAT`
3. Value: the token you just copied.

## Notes

- Keep `src/index.js` small and intentionally incomplete so POC Jira tickets have obvious, easy-to-verify things to add. A function that returns a string (not just `console.log`) makes it easier to write a test for the change.
- Do NOT add a `.gitignore` for `node_modules/` — there are no `node_modules/` since there are no npm dependencies. Add one only if a dependency is added later.
- The `node --test` runner (built into Node 18+) is used to avoid adding any dev dependencies to the sample app. Claude's `npm test` step will use this automatically.
