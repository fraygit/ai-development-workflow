# POC Epic — Task Breakdown

**Goal:** Prove the core AIDevFlow concept with zero platform infrastructure. A GitHub Actions workflow + a Claude Code skill file + a thin Jira webhook bridge — nothing else.

**POC repo:** `ai-development-workflow-poc/` ([fraygit/ai-development-workflow-poc](https://github.com/fraygit/ai-development-workflow-poc))  
**Sample app repo:** `ai-development-workflow-sample-nodejs` ([fraygit/ai-development-workflow-sample-nodejs](https://github.com/fraygit/ai-development-workflow-sample-nodejs)) — the target repo Claude makes changes to

**What the POC skips deliberately:** Coordination API, Webhook Receiver, Compiler, Prisma, Redis, Pub/Sub, multi-tenancy, billing, Terraform — every platform concern. This is a direct wire from Jira → GitHub Actions → Claude Code CLI.

**What the POC must prove:**
1. A Jira ticket assignment can trigger a GitHub Actions workflow (via a thin bridge).
2. Claude Code CLI can read a Jira ticket, implement code changes, and commit to a feature branch.
3. A SKILL.md file can reliably guide Claude's behaviour as a specific engineering persona.
4. A PR is created automatically with no human in the implementation loop.
5. (Story 2) Claude can post structured analysis back to Jira with no git operations.

## GitHub Repo Secrets Required

Configure all of these in `Settings → Secrets → Actions` in the **POC repo** (`ai-development-workflow-poc`):

| Secret | Description | Required for |
|---|---|---|
| `ANTHROPIC_API_KEY` | Anthropic API key — used by Claude Code CLI | Both stories |
| `JIRA_BASE_URL` | e.g. `https://yourorg.atlassian.net` | Both stories |
| `JIRA_API_TOKEN` | Jira API token (personal or bot account) | Both stories |
| `JIRA_USER_EMAIL` | Email paired with the Jira API token | Both stories |
| `SAMPLE_NODEJS_PAT` | Fine-grained PAT for `ai-development-workflow-sample-nodejs` — see Task 0.1 for scope | Story 1 |

`GITHUB_TOKEN` is **not used** for Story 1 — all sample repo operations (clone, push, PR) use `SAMPLE_NODEJS_PAT`. `GITHUB_TOKEN` is auto-provided but scoped only to the POC repo.

## Local Dev Prerequisites

1. Create and push the sample app: complete **Task 0.1** first.
2. Clone the POC repo: `git clone https://github.com/fraygit/ai-development-workflow-poc.git ai-development-workflow-poc`
3. Install Claude Code CLI locally to test skills: `npm install -g @anthropic-ai/claude-code`
4. Set `ANTHROPIC_API_KEY` in your local environment for manual skill testing.
