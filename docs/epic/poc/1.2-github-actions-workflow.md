# Task 1.2 — GitHub Actions Workflow (jira-implement.yml)

**Story:** 1.0 — Jira → Implement → PR  
**Skill:** senior-devops  
**Priority:** P0 — depends on Tasks 0.1 and 1.1

---

## What to do

Write `.github/workflows/jira-implement.yml` in the POC repo. This workflow:
1. Accepts a trigger from `repository_dispatch` (Jira bridge) or `workflow_dispatch` (manual test).
2. Checks out the POC repo (for skill files) and clones the sample app repo (the target) separately using a PAT.
3. Pre-fetches the Jira ticket.
4. Creates a feature branch in the sample app repo.
5. Runs Claude Code CLI with the `nodejs-developer` skill, working inside the sample app directory.
6. Pushes the branch and opens a PR — both using the PAT, not `GITHUB_TOKEN`.

The two repos serve distinct roles: the **POC repo** provides the skill; the **sample app repo** is what gets modified.

## File to Create

**Path in POC repo:** `.github/workflows/jira-implement.yml`

```yaml
name: Jira — Implement Ticket

on:
  repository_dispatch:
    types: [jira-ticket-assigned]
  workflow_dispatch:
    inputs:
      jira_ticket_id:
        description: 'Jira ticket ID (e.g. POC-1)'
        required: true
        type: string

jobs:
  implement:
    runs-on: ubuntu-latest
    # No special permissions needed — all sample repo operations use SAMPLE_NODEJS_PAT

    steps:
      - name: Checkout POC repo (for skill files)
        uses: actions/checkout@v4
        with:
          path: poc

      - name: Resolve Jira ticket ID
        id: ticket
        run: |
          if [ "${{ github.event_name }}" = "repository_dispatch" ]; then
            echo "id=${{ github.event.client_payload.ticket_id }}" >> $GITHUB_OUTPUT
          else
            echo "id=${{ inputs.jira_ticket_id }}" >> $GITHUB_OUTPUT
          fi

      - name: Clone sample Node.js app
        env:
          SAMPLE_NODEJS_PAT: ${{ secrets.SAMPLE_NODEJS_PAT }}
        run: |
          git clone \
            https://x-access-token:$SAMPLE_NODEJS_PAT@github.com/fraygit/ai-development-workflow-sample-nodejs.git \
            target

      - name: Set up Node.js 20
        uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Install sample app dependencies
        working-directory: target
        run: npm ci

      - name: Fetch Jira ticket
        id: jira
        env:
          JIRA_BASE_URL: ${{ secrets.JIRA_BASE_URL }}
          JIRA_API_TOKEN: ${{ secrets.JIRA_API_TOKEN }}
          JIRA_USER_EMAIL: ${{ secrets.JIRA_USER_EMAIL }}
        run: |
          TICKET_ID="${{ steps.ticket.outputs.id }}"
          AUTH=$(echo -n "$JIRA_USER_EMAIL:$JIRA_API_TOKEN" | base64 -w0)

          RESPONSE=$(curl -sf \
            -H "Authorization: Basic $AUTH" \
            -H "Content-Type: application/json" \
            "$JIRA_BASE_URL/rest/api/3/issue/$TICKET_ID")

          SUMMARY=$(echo "$RESPONSE" | jq -r '.fields.summary')
          DESCRIPTION=$(echo "$RESPONSE" | jq -r '
            [.fields.description.content[]?.content[]?
              | select(.type == "text") | .text] | join(" ")' 2>/dev/null || echo "")

          echo "summary=$SUMMARY" >> $GITHUB_OUTPUT
          {
            echo "description<<EOF"
            echo "$DESCRIPTION"
            echo "EOF"
          } >> $GITHUB_OUTPUT

      - name: Create feature branch
        id: branch
        working-directory: target
        run: |
          BRANCH="feature/${{ steps.ticket.outputs.id }}-ai-implementation"
          git checkout -b "$BRANCH"
          git config user.name "AIDevFlow Bot"
          git config user.email "bot@aidevflow.io"
          echo "name=$BRANCH" >> $GITHUB_OUTPUT

      - name: Install Claude Code CLI
        run: npm install -g @anthropic-ai/claude-code

      - name: Run nodejs-developer skill
        working-directory: target
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          JIRA_TICKET_ID: ${{ steps.ticket.outputs.id }}
          JIRA_SUMMARY: ${{ steps.jira.outputs.summary }}
          JIRA_DESCRIPTION: ${{ steps.jira.outputs.description }}
        run: |
          claude --no-interactive \
            --allowedTools "Read,Edit,Write,Bash,Glob,Grep" \
            -p "$(cat ../poc/.claude/skills/nodejs-developer/SKILL.md)"

      - name: Check for changes
        id: changes
        working-directory: target
        run: |
          if git diff --quiet && git diff --staged --quiet; then
            echo "has_changes=false" >> $GITHUB_OUTPUT
          else
            echo "has_changes=true" >> $GITHUB_OUTPUT
          fi

      - name: Fail if no changes made
        if: steps.changes.outputs.has_changes == 'false'
        run: |
          echo "Claude made no file changes. Check the skill output above."
          exit 1

      - name: Push branch to sample app repo
        working-directory: target
        env:
          SAMPLE_NODEJS_PAT: ${{ secrets.SAMPLE_NODEJS_PAT }}
        run: |
          git push \
            https://x-access-token:$SAMPLE_NODEJS_PAT@github.com/fraygit/ai-development-workflow-sample-nodejs.git \
            "${{ steps.branch.outputs.name }}"

      - name: Create Pull Request
        working-directory: target
        env:
          GH_TOKEN: ${{ secrets.SAMPLE_NODEJS_PAT }}
          JIRA_BASE_URL: ${{ secrets.JIRA_BASE_URL }}
        run: |
          TICKET_ID="${{ steps.ticket.outputs.id }}"
          BRANCH="${{ steps.branch.outputs.name }}"

          gh pr create \
            --title "[$TICKET_ID] AI Implementation" \
            --body "## AI-generated implementation

**Jira ticket:** $JIRA_BASE_URL/browse/$TICKET_ID

This PR was generated by Claude Code (nodejs-developer skill). Review all changes carefully before merging.

> ⚠️ **Auto-merge is disabled.** A human must review and approve this PR before it can be merged." \
            --head "$BRANCH" \
            --base main \
            --repo fraygit/ai-development-workflow-sample-nodejs
```

## Notes

- **Why `x-access-token` in the clone URL?** This is the standard way to embed a PAT in a git clone URL for GitHub. The token is masked in the Actions log. The push step uses the same pattern — no separate `git remote set-url` needed.
- **`working-directory: target`** on the Claude step is what ensures Claude edits the sample app, not the POC repo. All file tool calls from Claude resolve relative to that directory.
- **`--repo` flag on `gh pr create`** is required because the `gh` CLI detects the current git remote to determine where to create the PR, but we want to be explicit since there are two repos in the workspace.
- **No `contents: write` permission at the job level** — the job-level permissions only affect `GITHUB_TOKEN` (the POC repo token). Since we never write to the POC repo in this workflow, no elevated permissions are needed.
- **`npm ci` vs `npm install`** — `npm ci` is used for the sample app because it is deterministic and fast. If the sample app has no `package-lock.json`, change this to `npm install` for the initial test.
